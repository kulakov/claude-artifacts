# Proposal: Module 3 — Advanced Claude Code Features

## Концепция

**Level 3** фокусируется на кастомизации и автоматизации Claude Code для построения персонального "Product OS".

**Целевая аудитория:** PM который прошёл Level 1-2 и хочет автоматизировать повторяющиеся задачи.

---

## Предлагаемые модули

### 3.1 Custom Slash Commands
**Цель:** Создание собственных команд для частых задач

**Чему научится:**
- Структура slash commands (`.claude/commands/`)
- Создание команды с параметрами
- Практика: создать команду `/daily-standup` для генерации standup notes

**Материалы:**
- Примеры команд
- Template для создания команд

**Упражнение:**
1. Создать `/weekly-report` — генерирует weekly update из git commits
2. Создать `/competitor-check [company]` — быстрый анализ конкурента
3. Создать `/prd-review` — чеклист ревью PRD

---

### 3.2 Project Memory (CLAUDE.md Advanced)
**Цель:** Настройка Claude под конкретный проект

**Чему научится:**
- Структура CLAUDE.md
- Project-specific instructions
- Persona и tone настройки
- Include/exclude файлов

**Материалы:**
- Примеры CLAUDE.md для разных проектов
- Best practices guide

**Упражнение:**
1. Создать CLAUDE.md для TaskFlow с контекстом, персонами, правилами
2. Добавить project-specific команды
3. Настроить tone и язык (русский)

---

### 3.3 Hooks & Automation
**Цель:** Автоматизация через hooks

**Чему научится:**
- Что такое hooks в Claude Code
- Pre-tool и Post-tool hooks
- Интеграция с git workflow

**Материалы:**
- Примеры hooks
- Use cases для PM

**Упражнение:**
1. Hook на автоматический формат markdown
2. Hook на проверку TODO перед commit
3. Hook на генерацию changelog

---

### 3.4 MCP Servers (Integrations)
**Цель:** Подключение внешних сервисов

**Чему научится:**
- Что такое MCP (Model Context Protocol)
- Установка MCP серверов
- Практика: подключить Notion/Linear/GitHub

**Материалы:**
- Список полезных MCP серверов для PM
- Инструкции по установке

**Упражнение:**
1. Подключить MCP для работы с файловой системой
2. Настроить интеграцию с task tracker
3. Создать workflow: идея → PRD → tasks

---

## Альтернативные идеи для модулей
