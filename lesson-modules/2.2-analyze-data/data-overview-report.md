# Обзор доступных данных для анализа активации TaskFlow

**Дата:** Декабрь 2024
**Контекст:** Activation rate застрял на 45%, анализируем причины и решения

---

## Датасеты в нашем распоряжении

### 1. `activation-funnel-q4.csv`

**Источник:** Mixpanel (агрегированный экспорт воронки)
**Записей:** 4 строки (шаги воронки)

| Поле | Описание | Пример |
|------|----------|--------|
| `step` | Название шага воронки | "First Task Created" |
| `users_entered` | Сколько вошло на шаг | 10,000 |
| `users_completed` | Сколько завершило | 7,200 |
| `completion_rate` | Конверсия | 0.72 |
| `median_time_to_complete` | Медианное время (мин) | 18 |

**Ограничения:** Только агрегаты, нет разбивки по пользователям или сегментам.

---

### 2. `user-survey-responses.csv`

**Источник:** Опрос недавних регистраций
**Записей:** 800 респондентов

| Поле | Описание | Значения |
|------|----------|----------|
| `user_id` | ID респондента | survey_user_0001 |
| `company_size` | Размер компании | "5-20", "21-99", "100+" |
| `onboarding_rating` | Оценка онбординга | 2, 3, 4 |
| `biggest_confusion` | Главная проблема (открытый текст) | "Stared at empty project..." |
| `feature_request` | Запрос фичи | "Templates or examples", "Simpler onboarding", "Better help docs", "Sample projects or templates" |
| `would_recommend` | Рекомендовал бы? | "Yes", "No", "Maybe" |

**Ограничения:** Нет поля "предыдущий инструмент", нет времени использования, нет связи с поведением в продукте.

---

### 3. `taskflow-usage-data-q4.csv`

**Источник:** Event-level данные из продукта
**Записей:** Тысячи событий (event log)

| Поле | Описание | Значения |
|------|----------|----------|
| `user_id` | ID пользователя | user_0001 |
| `event_type` | Тип события | "signup", "first_task_created", "first_task_completed", "invite_sent" |
| `timestamp` | Время события | 2024-10-27T16:51:53 |
| `company_size` | Размер компании | "5-20", "21-99", "100+" |
| `user_role` | Роль пользователя | "PM", "Engineer", "Designer", "Founder", "Manager" |
| `industry` | Индустрия | "Marketing", "SaaS", "FinTech", "E-commerce" |

**Возможности:** Можно посчитать time-to-value, сегментировать по роли/индустрии, построить когортный анализ.

---

### 4. `onboarding-experiment-results.csv`

**Источник:** A/B тест из LaunchDarkly
**Записей:** 8,000 пользователей (4,000 control + 4,000 treatment)

| Поле | Описание | Значения |
|------|----------|----------|
| `user_id` | ID пользователя | control_user_0000, treatment_user_0000 |
| `cohort` | Группа эксперимента | "control", "treatment" |
| `signup_date` | Дата регистрации | 2024-11-06 |
| `company_size` | Размер компании | "5-20", "21-99", "100+" |
| `user_role` | Роль | "PM", "Engineer", "Designer", "Founder", "Manager" |
| `completed_first_task` | **PRIMARY METRIC** — активация | True/False |
| `time_to_first_task_minutes` | Время до первой задачи | 28, 51, пусто если не активировался |
| `invited_teammate` | Пригласил коллегу | True/False |
| `used_task_template` | Использовал шаблон | True/False |
| `days_active_week_1` | Дней активности в неделю 1 | 0-7 |
| `tasks_completed_week_1` | Задач за неделю 1 | 0, 1, 2, 4... |

**Возможности:** Полный анализ эксперимента — сегментация, качество активации, leading indicators.

---

## Как данные связаны

```
activation-funnel-q4.csv     → Агрегированная воронка (WHERE проблема)
         ↓
user-survey-responses.csv    → Качественные причины (WHY проблема)
         ↓
taskflow-usage-data-q4.csv   → Детальное поведение (для impact estimation)
         ↓
onboarding-experiment-results.csv → Результаты A/B теста (DID решение сработало)
```

## Чего НЕТ в данных

| Отсутствует | Было бы полезно для |
|-------------|---------------------|
| Предыдущий инструмент (Asana, Jira...) | Понять ожидания мигрантов |
| Tenure / время использования | Сегментация новички vs опытные |
| Платный vs бесплатный | Связь с монетизацией |
| Причина отмены (churned users) | Понять почему уходят |
| Кол-во задач в аккаунте | Связь активации с объёмом работы |

## Ключевые находки из анализа

### Воронка активации

| Шаг | Вошло | Завершило | Конверсия | Медианное время |
|-----|-------|-----------|-----------|-----------------|
| Signup | 10,000 | 10,000 | 100% | 0 мин |
| First Task Created | 10,000 | 7,200 | 72% | 18 мин |
| First Task Completed | 7,200 | 2,880 | **40%** | 45 мин |
| Invite Sent | 2,880 | 1,440 | 50% | 24 мин |

**Главная проблема:** 60% пользователей создают задачу, но не завершают её.

### Анализ опроса (800 респондентов)

**Топ жалоб:**

| Кластер проблемы | % респондентов |
|------------------|----------------|
| "Не знаю что создавать" | 35% |
| "Нужны примеры/шаблоны" | 28% |
| "Пустой проект пугает" | 22% |
| Проблемы с интерфейсом | 15% |

**Сегментация по размеру компании:**

| Размер | % респондентов | "Не знаю что создавать" |
|--------|----------------|------------------------|
| 5-20 (маленькие) | 60% | **42%** |
| 21-99 (средние) | 30% | 28% |
| 100+ (enterprise) | 10% | 18% |

**Вывод:** Маленькие команды страдают от "пустого проекта" в 2.3 раза чаще enterprise.

## Рекомендации по сбору данных

Для более глубокого анализа в будущем рекомендуется добавить:

1. **В форму регистрации:** "Какой инструмент использовали раньше?"
2. **В опрос:** Связь с user_id для корреляции с поведением
3. **В продукт:** Трекинг количества задач в момент drop-off
4. **В churn survey:** Причины отмены подписки
