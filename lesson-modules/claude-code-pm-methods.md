# МОДУЛЬ 2.1: НАПИСАНИЕ PRD

## @-mentions (Контекстные ссылки)

**Что это:** Подключение файлов и контекста к разговору через @

**Как использовать:**
```
@company-context/PRODUCT.md — подключить контекст продукта
@templates/prd-template.md — использовать шаблон
@user-research/interviews.md — добавить исследования
```

**Когда применять:**
- Написание PRD с контекстом компании
- Использование шаблонов документов
- Подключение user research к анализу

---

## Параллельные агенты

**Что это:** Запуск нескольких AI-агентов одновременно для генерации вариантов

**Как использовать:**
```
"Запусти 3 агента для генерации 3 разных подходов к PRD"
```

**Когда применять:**
- Генерация альтернативных решений
- Исследование разных стратегических направлений
- Создание вариантов для выбора

**Преимущества:**
- Экономия времени (параллельно, не последовательно)
- Разнообразие подходов
- Возможность выбрать лучший вариант

---

## Multi-perspective Review (Sub-agents)

**Что это:** Получение feedback от разных "персон" через специализированных агентов

**Доступные агенты:**
| Агент | Фокус | Когда использовать |
|-------|-------|-------------------|
| **Engineer** | Техническая feasibility | Проверка реализуемости |
| **Executive** | Бизнес-impact | Alignment со стратегией |
| **User Researcher** | User needs | Валидация с потребностями |

**Как использовать:**
```
"Получи ревью от engineer, executive и user researcher агентов"
```

**Когда применять:**
- Финальная проверка PRD
- Валидация стратегических решений
- Поиск blind spots в документах

# МОДУЛЬ 2.2: АНАЛИЗ ДАННЫХ

## CSV/Data Analysis

**Что это:** Анализ данных из CSV файлов через Claude Code

**Как использовать:**
```
"Проанализируй воронку активации из activation-funnel.csv"
"Найди drop-off между шагами"
```

**Типичные задачи:**
- Анализ воронок конверсии
- Поиск drop-off точек
- Расчёт метрик (conversion rates, retention)

---

## Сегментация данных

**Что это:** Разбивка данных по сегментам для deeper insights

**Как использовать:**
```
"Сегментируй результаты по размеру компании"
"Разбей данные по ролям пользователей"
```

**Типы сегментации:**
- По размеру компании (SMB, Mid-market, Enterprise)
- По роли (PM, Engineer, Designer, Manager)
- По индустрии
- По cohort/времени

**Ключевой инсайт:** Topline метрики часто маскируют segment wins/losses

---

## Survey Data Processing

**Что это:** Анализ качественных данных из опросов

**Как использовать:**
```
"Кластеризуй ответы по темам"
"Найди топ причины проблем"
```

**Техники:**
- Кластеризация открытых ответов
- Подсчёт частоты тем
- Сегментация по демографии респондентов

---

## ROI Моделирование

**Что это:** Построение финансовых моделей для оценки impact фичей

**Формула базового impact:**
```
Impact = Users Affected × Current Rate × Expected Lift × Value per Action
```

**Три сценария:**
| Сценарий | Описание | Использование |
|----------|----------|---------------|
| Pessimistic | Всё идёт плохо | Minimum viable case |
| Realistic | Ожидаемый результат | Planning baseline |
| Optimistic | Всё идёт отлично | Upside potential |

**Когда применять:**
- Приоритизация фич
- Защита инвестиций перед leadership
- Оценка engineering estimates

---

## A/B Test Analysis

**Что это:** Анализ результатов экспериментов

**Ключевые метрики:**
- Primary metric (activation, conversion)
- Quality metrics (retention, engagement)
- Leading indicators (предсказывают долгосрочный успех)

**Процесс анализа:**
1. Topline результаты
2. Сегментация по ключевым dimensions
3. Quality metrics для активированных
4. Leading indicators
5. Рекомендация (ship/iterate/kill)

**Важно:** Всегда сегментируй — topline может скрывать wins/losses

# МОДУЛЬ 2.3: ПРОДУКТОВАЯ СТРАТЕГИЯ

## Competitive Research (Parallel Agents)

**Что это:** Быстрое исследование конкурентов через параллельных агентов

**Как использовать:**
```
"Запусти агентов для исследования AI-стратегий Notion, Linear, Asana"
```

**Что исследовать:**
- Запущенные фичи
- Ценообразование и positioning
- Целевая аудитория
- Стратегическое направление

**Результат:** Competitive landscape за минуты вместо часов

---

## Rumelt's Strategy Kernel

**Что это:** Фреймворк для разработки стратегии

**Три компонента:**

### 1. ДИАГНОЗ
> "Что на самом деле происходит?"

- Идентифицирует ядро проблемы/возможности
- Основан на данных и competitive analysis
- Упрощает сложность до essential issue

### 2. РУКОВОДЯЩАЯ ПОЛИТИКА
> "Какой наш подход?"

- Делает жёсткие выборы о WHERE to compete
- Говорит НЕТ некоторым возможностям
- Создаёт преимущество через tradeoffs

### 3. СОГЛАСОВАННЫЕ ДЕЙСТВИЯ
> "Как мы исполняем?"

- Конкретные инициативы, усиливающие друг друга
- Sequenced roadmap с dependencies
- Ресурсы сфокусированы на стратегии

**Красные флаги плохой стратегии:**
- ❌ Цели вместо стратегии ("увеличить выручку")
- ❌ Нет tradeoffs ("сделаем всё для всех")
- ❌ Несогласованные действия (список без связей)

---

## Devil's Advocate

**Что это:** Техника pressure-testing решений

**Как использовать:**
```
"Сыграй адвоката дьявола — челленджи мой выбор"
```

**Процесс:**
1. Принять решение
2. Получить challenge с контраргументами
3. Либо защитить решение, либо пересмотреть
4. Финальное решение более robust

**Когда применять:**
- Стратегические выборы
- Крупные product decisions
- Перед презентацией leadership

## Skills (Трансформация документов)

**Что это:** Специализированные capabilities для создания файлов

**Доступные skills:**
| Skill | Результат | Применение |
|-------|-----------|------------|
| **pptx** | PowerPoint | Executive presentations |
| **xlsx** | Excel | Финансовые модели, данные |
| **pdf** | PDF | Формальные документы |
| **docx** | Word | Отчёты, спецификации |

**Как использовать:**
```
"Создай слайды из моей стратегии"
```

**Когда применять:**
- Презентации для leadership
- Sharing документов с stakeholders
- Формальные deliverables

# КЛЮЧЕВОЙ ПРИНЦИП

> **AI не заменяет PM-мышление. AI усиливает его.**

- PRD — ты выбираешь подход и принимаешь feedback
- Данные — ты задаёшь вопросы и интерпретируешь
- Стратегия — ты делаешь tradeoffs и защищаешь решения

AI ускоряет исследования, генерацию вариантов, stress-testing.
Но judgment остаётся твоим.
