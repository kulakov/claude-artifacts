# PRD Review: AI Context-First помощник для управления задачами

**Документ:** ai-chat-todo-prd-v2.md (Context-First подход)
**Дата ревью:** Декабрь 2024

---

## Engineering Review

**Вердикт:** Технически выполнимо, но timeline оптимистичен

### Ключевые замечания:

1. **Большой разрыв между амбициями и реальностью на Phase 0-1**
   - Требуется fine-tuned ML модель на истории пользователя, но Phase 0 всего 2 недели
   - Для достижения 75% accuracy на контексте нужны качественные данные и итерации
   - Риск: Phase 1 может затянуться из-за недостаточной базы для обучения модели

2. **Голосовой ввод — потенциальный blocker для русского языка**
   - Whisper API хорошо работает на EN, но для RU + смешанный RU/EN ввод нужно тестировать
   - Рекомендация: выделить дополнительный sprint на языковое тестирование в Phase 0

3. **Сложность context engine недооценена**
   - "Анализ контекста <1 секунда" + vector embeddings + semantic search = 3+ разных компонента
   - Нет деталей о том, как система обновляет контекст при переходе между проектами

4. **Метрики амбициозны для MVP**
   - "<15% негативных отзывов" — размыто. Нужна четкая метрика: % tasks где user быстро исправил prediction

5. **Инфраструктурные и compliance вопросы не решены**
   - GDPR: consent flow, retention policy, deletion API не описаны
   - Нет fallback если OpenAI API down

**Рекомендация:** Добавить 2 недели буфера для языкового тестирования и уточнить архитектуру context engine.

## Executive Review

**Вердикт:** ОДОБРЕНО С ЗАМЕЧАНИЯМИ

### Ключевые замечания:

1. **Бизнес-ценность солидная, но ROI требует доказательства**
   - "+30% задач" влияет на engagement и retention — наша боль в Q1
   - Но слабо описана связь между ресурсами (~$30k OpenAI API) и revenue/churn impact
   - Нужен бизнес-кейс: "Если +30% задач удерживает 5% оттока, это X тыс. ARR saved"

2. **Стратегическое соответствие — сильно**
   - Race с Asana/Linear по AI, контекстный подход — реальный дифференциатор
   - Опасность: если Phase 1 не даст 40%+ adoption, мы потратили Q1 впустую
   - Нужна жесткая go/no-go gate на 28 февраля

3. **Конкурентное позиционирование — острое, но переоценено**
   - "Контекст" абстрактно для customer. Нужно переформулировать для Sales:
   - "AI понимает твой workflow" вместо "контекстный анализ"

4. **Критические риски:**
   - ML accuracy <70% — если модель не работает, теряем месяц. Начать rule-based PoC параллельно
   - Voice для русского языка — contingency: English-only launch сначала

**Next step:** Согласовать accuracy thresholds с CTO перед Phase 0 kickoff.

## User Researcher Review

**Вердикт:** PRD стратегически сильный, но UX decisions опираются на непроверенные предположения

### Ключевые замечания:

1. **Gap: отсутствие валидации pain point про "страх пустой формы"**
   - Это предположение PM, а не подтвержденный паттерн
   - Нужно измерить: что именно пользователей "замораживает"?
   - Рекомендация: User Interview на 5-7 пользователях перед разработкой

2. **Риск юзабилити: confidence indicators могут запутать**
   - Система ✓/✓/⚠️ выглядит логично, но эффективность не доказана
   - Что делает пользователь с 65% уверенностью? Сколько тапов нужно?
   - Рекомендация: Usability Testing (5-8 пользователей) на task preview экране

3. **Нет типологии пользователей**
   - "PM в remote команде" слишком широко
   - 45% "задач не создаются" — это глобальная метрика или специфичный сегмент?
   - Рекомендация: Segmentation Analysis перед Phase 1

4. **UX ловушка: voice может снизить accuracy**
   - Риск: пользователь быстро подтвердит preview, даже если 1-2 поля неправильные
   - Рекомендация: измерять % voice-задач, требующих коррекции в течение 5 минут

5. **Missing: понимание реальных ситуаций использования**
   - Может, людям нужна 2-секундная задача (пусть неидеальная), а не 8-секундная гладкая flow
   - Рекомендация: Contextual Inquiries — наблюдение где/когда/как люди реально создают задачи
