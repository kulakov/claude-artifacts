# Стиль коммуникации: Notion Document

## Цель
Комплексный справочный документ, который любой может прочитать асинхронно и получить полный контекст.

## Правила форматирования
- **Длина:** Настолько длинно, насколько нужно для тщательности (обычно 300-600 слов)
- **Тон:** Профессиональный но доступный, детальный, хорошо организованный
- **Структура:**
  - **Заголовок:** Чёткий, описательный headline
  - **Резюме:** 2-3 предложения TL;DR в начале
  - **Разделы:** Используйте H2 заголовки для организации (Бэкграунд, Ключевые находки, Рекомендации, Следующие шаги)
  - **Списки:** Буллет-поинты для лёгкости сканирования
  - **Таблицы:** Опционально для организации данных/находок
  - **Callouts:** Используйте для важных инсайтов или action items
- **Включить:** Полный контекст, подтверждающие детали, прямые цитаты, конкретные примеры
- **Избегать:** Предположения о предварительных знаниях, пропуск важного контекста
- **Фокус:** Быть комплексным, самостоятельным справочником

## Пример

**Входные данные:** Пользовательские исследования показали, что 8/8 пользователей испытывали трудности с онбордингом, тратя 45+ минут на выполнение первой задачи. Основные проблемы: непонятная навигация (6 упоминаний), нет шаблонов (5 упоминаний), подавляющий пустой экран (4 упоминания). Рекомендовано: добавить интерактивный тур и библиотеку шаблонов.

**Результат:**

# Отчёт по пользовательским исследованиям: Исследование онбординга TaskFlow

**TL;DR:** Все 8 пользователей значительно испытывали трудности с онбордингом (в среднем 45+ мин до первой задачи). Основные проблемы: запутанность навигации, отсутствие шаблонов, подавляющий опыт пустого экрана. Рекомендуем интерактивный тур + библиотеку шаблонов на Q1.

## Бэкграунд

Мы провели 8 интервью с пользователями (2-6 октября) с новыми пользователями TaskFlow для понимания проблемных точек онбординга. Участники были набраны из недавних trial signups и представляли микс ролей (PM, инженеры, маркетологи) и размеров команд (от solo до 50+ человек).

**Цели исследования:**
- Выявить проблемные точки в опыте первого использования
- Понять time-to-first-task completion
- Сравнить наш онбординг с опытом конкурентов
- Сгенерировать рекомендации для улучшения активации в Q1

## Ключевые находки

### 1. Time-to-Value слишком долгое
- **Среднее время выполнения первой задачи: 45 минут**
- Все 8 пользователей (100%) испытывали трудности с начальной настройкой
- Конкуренты показывают в среднем 15 минут (Linear, Asana)
- Текущий показатель активации: 45% (цель: 60%)

### 2. Навигация запутанная (6/8 пользователей)
Пользователи не могли найти базовые функции:
- "Я не знал, куда кликнуть чтобы создать свой первый проект"
- "Я искал шаблоны но не мог их найти"
- "Боковая панель не была очевидна как навигация"

### 3. Отсутствие шаблонов увеличивает трение (5/8 пользователей)
Пользователи хотят готовые стартовые точки:
- "Я смотрел на пустой экран не зная что делать"
- "Я бы хотел иметь примеры, с которых можно начать"
- "В Notion есть шаблоны — почему у вас нет?"

### 4. Опыт пустого экрана подавляет (4/8 пользователей)
Пустое рабочее пространство пугает новых пользователей:
- "Я чувствовал, что должен строить всё с нуля"
- "Это слишком много свободы без руководства"

## Рекомендации

### Приоритет: Интерактивный продуктовый тур
- Проведёт пользователей через ключевые функции при первом входе
- Progressive disclosure (не перегружать)
- Можно пропустить, но по умолчанию "включен"
- Оценка инженерной команды: 2 спринта

### Приоритет: Библиотека шаблонов
- 5-7 готовых шаблонов проектов
- Категории: Product Launch, Sprint Planning, Bug Tracking, Content Calendar
- Показывается сразу после регистрации
- Оценка инженерной команды: 3 спринта (web), 1 спринт (mobile позже)

### Второстепенное: Улучшения навигации
- Более заметные метки боковой панели
- Контекстные подсказки для новых пользователей
- Включено в интерактивный тур
